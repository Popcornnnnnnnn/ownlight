# 2026-05-30 Simulator UI Audit

## 背景

本轮审计使用 `Private Moments iPhone 13 Pro` 模拟器和 demo data，截图证据保存在：

`.tmp/ui-review/global-ui-audit-20260530/`

目标不是立刻改代码，而是把全局 UI 中值得优化的点记录下来，后续逐条处理、逐条验证。

## 审计清单

### 1. 顶部工具栏的可访问性和操作语义

状态：已处理，2026-05-30

白话解释：现在 Timeline、Calendar、Check-ins 顶部那些只有图标的按钮，视觉上能看到，但系统层面不够像“明确的按钮”。模拟器里底部 Tab 可以被识别为 `Timeline`、`Calendar`、`Check-ins`，但顶部搜索、筛选、设置、写入、统计、管理等图标很难被稳定识别和操作。

这不是说用户肉眼看不到按钮，而是说 iOS 的辅助功能和 UI 自动化工具不一定知道这些图标分别是什么。对真实用户的影响是 VoiceOver 可能读不清按钮含义；对开发验证的影响是以后很难写可靠的 UI 测试去点击这些入口。

建议方向：

- 抽一个统一的顶部图标按钮组件，例如 `TopActionButton` / `TopActionBar`。
- 每个按钮补清晰的 `accessibilityLabel`，例如 `Search`、`Filter`、`Settings`、`New Moment`、`Month Stats`、`Manage Check-ins`。
- 每个关键按钮补稳定的 `accessibilityIdentifier`，例如 `timeline.searchButton`、`calendar.monthStatsButton`。
- 统一最小点击区域，避免图标看得见但点起来不稳定。

验收标准：

- 用 Accessibility Inspector 或 UI automation 能看到每个顶部图标按钮的明确名称。
- 自动化脚本能稳定打开 Settings、Month Stats、Composer、Check-ins Manage 等入口。
- 肉眼 UI 不需要明显变化，主要是底层语义和可测试性变好。

实现记录：

- 新增 `TopActionButton` / `TopActionMenuLabel`，统一顶部图标按钮的最小点击区域、accessibility label 和 accessibility identifier。
- 已接入 Timeline、Calendar、Check-ins 主页面顶部入口，以及 Check-ins Manage 的新增入口。
- 当前 identifier 包括 `timeline.searchButton`、`timeline.filterButton`、`timeline.settingsButton`、`timeline.newMomentButton`、`calendar.reviewsButton`、`calendar.filterMenu`、`calendar.settingsButton`、`calendar.monthStatsButton`、`checkIns.modeToggleButton`、`checkIns.manageButton`、`checkIns.manage.newButton`。
- 2026-05-30 修正：`TopActionButton` 不再在 toolbar 中占用 44pt 宽度；Timeline 和 Calendar 顶部入口保持和 Check-ins 一样的 `ToolbarItemGroup` 结构，避免自定义 `HStack` 过窄，同时保留 label/identifier。

### 2. 统一页面顶部节奏

状态：已处理，2026-05-30

Timeline、Calendar、Check-ins 使用超大标题加浮动胶囊工具栏；Settings 内页使用居中标题和很大的 `Edit` 按钮；Storage 页面又是另一种布局。建议主页面保留大标题和浮动工具栏，设置内页统一回到标准 inline navigation，减少每个页面各自发明一套顶部结构。

实现记录：

- Settings 内页继续使用 inline navigation；`Tags` 从大标题改为 inline 标题，和 `AI & Analysis`、`Storage & Export`、`Appearance`、`Language`、`Display` 保持一致。
- `AI & Analysis` 和 `Tags` 的右上角编辑入口从文字胶囊改成轻量图标按钮，保留 `Edit` / `Done` 的辅助功能名称，并补稳定 identifier：`aiAnalysis.editProvidersButton`、`tags.editButton`。

### 3. Settings 动态字号适配

状态：本轮不处理，2026-05-31

在 `accessibility-large` 字号下，Settings 首页的左右两列很快出现省略号，例如 `This iPhone`、`Apple Cloud`、`Language`。建议大字号时改为上下两行布局：标题在第一行，状态/value/badge 在第二行。

决策记录：

- 当前用户验收重点是常规字号下的视觉节奏；极端动态字号边缘情况暂不处理，避免 Settings 首页重新变成高低不齐的两行布局。

### 4. 底部浮动 Tab 遮挡 Timeline 内容

状态：已处理，2026-05-31

Timeline 底部内容会被浮动 Tab 遮住。建议给列表增加足够的 bottom safe area/padding，确保最后一条内容可以完整滚到 Tab 上方。

实现记录：

- Timeline 底部在没有评论输入条时增加透明 bottom safe-area inset，让最后一条内容可以滚到系统浮动 Tab 上方。

### 5. AI & Analysis 全局开关和子功能状态关系

状态：已处理，2026-05-31

当全局 AI 是 `Off` 时，`AI Title Auto-Insert` 仍然显示为开启，容易让人误以为它会继续生效。建议全局关闭时把 generated artifacts 区域整体 disabled，或增加明确状态说明。

实现记录：

- `Generated Artifacts` 区域在全局 AI 关闭时整体 disabled，并显示一行 footer 说明这些开关会在 AI Analysis 开启后生效。
- 不修改用户保存的子开关值，避免重新开启 AI 时丢失偏好。

### 6. Settings 图标色彩系统

状态：已处理，2026-05-31

Settings 首页彩色图标接近 iOS Settings，但和 Timeline/Calendar 的黑白极简风格不完全一致。建议降低饱和度，或只在一级入口使用彩色，内页回到单色系统图标。

实现记录：

- Settings 首页入口继续保留彩色图标作为一级导航信号，但统一降低色块透明度和白色图标强度，让整体更接近当前应用的轻量风格。

### 7. Calendar 月视图下半屏空白

状态：已处理，2026-05-31

Calendar 月历下方到 Tab 之间空白较多。可以加入轻量 selected-day preview、month stats 摘要，或者压缩月份标题和 grid 的垂直节奏。

实现记录：

- 月历下方新增一条轻量 month summary row，展示总数、活跃天数、最多的一天。
- 点击该行打开现有 `Month Stats` sheet，不新增第二套统计逻辑。

### 8. Check-ins History 横向筛选 chip 截断

状态：已处理，2026-05-31

Check-ins History 顶部横向 chip 列表右侧露出半个 chip，但滚动暗示不明显。建议加 trailing fade、更多右侧 padding，或收紧 chip 宽度。

实现记录：

- History filter bar 保持单行横向滚动，右侧增加 trailing fade 和滚动余量，强化“这里还能横向滑”的暗示。

### 9. Storage & Export 文案密度

状态：已处理，2026-05-31

Storage & Export 的说明文字正确，但首屏像文档页，灰色 footer 过长。建议首屏只保留一句风险提示，把更详细说明放入 disclosure/info 页面。

实现记录：

- Export footer 缩短为一句风险提示。
- 详细的导出内容、排除项说明移入 `Export Details` disclosure，保留隐私和凭证边界语义。

### 10. Tags 页面 Edit 按钮形态

状态：已处理，2026-05-30

Tags 内容列表可读性较好，但右上角 `Edit` 按钮和 Settings 内页一样显得过大。建议统一为标准 toolbar text button，减少浮层感。

实现记录：

- 随第 2 条合并处理，Tags 右上角编辑入口已改为轻量图标按钮，避免文字按钮在 iOS 26 上形成过重的胶囊。
